Recovery Audit

현재 포트폴리오 repo는 원본 코드의 한계를 숨기지 않고, 공개 가능한 상태로 만드는 1차 복구 작업을 반영합니다.

FindingImpactAction
로컬 폴더가 원격 이력과 분리됨잘못된 소스 기준으로 작업하면 기능 파일이 더 사라질 수 있음원격 원본 스냅샷을 기준으로 새 포트폴리오 repo에 이관
구버전 Lombok이 Java 21 javac에서 실패Maven compile 불가Lombok과 Maven compiler/war plugin 업데이트
DB, SMTP, Kakao 키 하드코딩공개 repo 보안 리스크현재 트리에서 더미 값과 가드 로직으로 치환
채팅 클라이언트가 과거 EC2 주소에 고정됨현재 서버 컨텍스트에서 WebSocket 기능이 동작하지 않음JSP context path 기준 /echo SockJS endpoint를 사용하도록 수정
target/ 산출물이 원본 커밋에 포함저장소 품질 저하, 불필요한 바이너리 확산포트폴리오 repo에서는 산출물 제외 및 `.gitignore` 정리
DB 스키마와 실행 데이터 미확정서버 앱 전체 기능을 즉시 시연하기 어려움GitHub Pages 정적 mock demo로 핵심 사용자 흐름 제공
