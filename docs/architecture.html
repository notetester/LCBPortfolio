Architecture

LCB Cinema는 공개 포트폴리오용 인터랙티브 Pages 화면과 Spring MVC WAR 서버 소스를 함께 담습니다. Pages 화면은 제품 흐름을 빠르게 검토하기 위한 프런트 시뮬레이터이고, 서버 소스는 JSP/MyBatis 기반 구현 구조를 보여줍니다.

Runtime Layers

LayerFilesResponsibility
Portfolio Appindex.html, portfolio/app.js, portfolio/app.css영화 탐색, 좌석 선점, 결제 승인, 티켓 취소, 스토어, 게시판, 채팅, 운영 콘솔을 GitHub Pages에서 즉시 검토할 수 있게 제공합니다.
Spring MVCsrc/main/java/com/lcb404/controller요청 매핑, 세션 기반 사용자 흐름, 화면 모델 구성, AJAX 응답을 담당합니다.
Servicesrc/main/java/com/lcb404/service회원, 영화, 예매, 게시판, 스토어 로직과 mapper 호출을 조합합니다.
Persistencesrc/main/resources/sqlmap, com.lcb404.mapperMyBatis mapper interface/XML로 SQL을 분리합니다. 사용자 입력은 parameter binding을 기준으로 처리합니다.
JSP Viewssrc/main/webapp/WEB-INF/views서버 렌더링 화면, JSTL, jQuery 기반 동작을 담당합니다.
RealtimeEchoHandler, show/trailer.jspSockJS/WebSocket endpoint를 통해 같이 보기 채팅 메시지를 broadcast합니다.

Request Flow

Browser
  -> CharacterEncodingFilter
  -> DispatcherServlet
  -> Login/Board Interceptor
  -> Controller
  -> Service
  -> MyBatis Mapper
  -> MySQL
  -> JSP View or JSON Response

Reservation Flow

GET /reserve/ticketSchedule
  -> showtime list

GET /reserve/theater?movieCode={id}
  -> branch and showtime selection

POST /reserve/seatReserve
  -> session user validation
  -> selected seat persistence
  -> timetable remaining seat update
  -> mypage ticket history

공개 데모에서는 같은 흐름을 클라이언트 상태로 시뮬레이션합니다. 좌석 선점 만료, 인원 수 검증, 결제 승인, 취소 시 좌석 반환을 화면에서 확인할 수 있습니다.

Chat Flow

Client SockJS
  -> /echo
  -> EchoHandler.afterConnectionEstablished
  -> EchoHandler.handleTextMessage
  -> broadcast to connected sessions
  -> EchoHandler.afterConnectionClosed

채팅 payload는 길이 제한과 로그 마스킹을 적용해 메시지 내용이 서버 로그에 그대로 남지 않도록 정리했습니다. 실제 서비스로 확장할 때는 roomId, senderId, moderation status를 별도 메시지 모델로 분리합니다.
