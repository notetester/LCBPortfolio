Domain Model

LCB Cinema는 영화관의 핵심 업무 흐름을 영화 탐색, 회차 선택, 좌석 선점, 결제 승인, 티켓 관리, 스토어 픽업, 고객 응대로 나누어 설계합니다.

Core Entities

EntityKey FieldsRule
MoviemovieId, title, grade, runtime, score, tags상영 가능 콘텐츠와 관람 등급 정보를 제공합니다.
BranchbranchId, screens회차는 반드시 하나의 지점과 상영관에 속합니다.
ShowtimeshowId, movieId, branchId, time, capacity, basePrice좌석 점유율, 조조/심야 정책, 판매 가능 좌석 계산의 기준입니다.
SeatHoldholdId, showId, seats, expiresAt결제 전 좌석을 4분간 임시 점유합니다. 만료되면 좌석을 반환합니다.
ReservationticketId, paymentKey, seats, status, refundUntil결제 승인 후 티켓이 생성되며 취소 시 좌석 점유를 해제합니다.
StoreOrderorderId, items, pickupBranch, status관람 전 스낵 픽업 흐름을 장바구니와 주문 상태로 분리합니다.
SupportPostpostId, type, status공지, Q&A, 분실물 접수 상태를 운영 콘솔에서 추적합니다.

Reservation State

AVAILABLE
  -> HELD        좌석 선택, 4분 임시 선점
  -> AVAILABLE   선점 만료 또는 사용자가 좌석 해제

HELD
  -> PAID        결제 승인, paymentKey 발급
  -> AVAILABLE   결제 실패 또는 선점 만료

PAID
  -> CANCELLED   상영 20분 전까지 취소/환불
  -> USED        입장 처리 후 취소 불가

Pricing Policy

ComponentCalculationPurpose
기본가showtime.basePrice * selectedSeats회차 포맷별 기준 가격입니다.
좌석 등급프리미엄 +3,000원, 스탠다드 0원, 이코노미 -1,000원상영관 좌석 등급별 차등 가격입니다.
시간 정책조조 -2,000원, 심야 +1,000원회차 시간대별 수요를 반영합니다.
멤버십소계의 5% 할인회원 예매 흐름을 보여줍니다.
수수료예매 건당 1,000원최종 결제 금액 계산을 분리합니다.

Failure Cases

CaseExpected Handling
인원 수와 좌석 수 불일치예매 확정 전 검증 메시지를 표시하고 티켓을 만들지 않습니다.
좌석 선점 만료선택 좌석을 비우고 다시 선택하도록 안내합니다.
동일 좌석 중복 예매확정 티켓의 좌석은 판매 완료 상태로 표시하고 재선택을 막습니다.
취소/환불티켓 상태를 취소 완료로 바꾸고 운영 콘솔의 활성 티켓과 매출에서 제외합니다.
스토어 빈 장바구니 결제주문을 생성하지 않고 상품 선택을 요청합니다.
